III. Primitive types and String

 

Remember those building blocks we discussed earlier? Well, here they are. Primitive types are designed to hold a certain type of data. In Java we have the following primitive types.

To store numbers :

byte: The byte data type is an 8-bit signed two's complement integer. It has a minimum value of -128 and a maximum value of 127 (inclusive)

short: The short data type is a 16-bit signed two's complement integer. It has a minimum value of -32,768 and a maximum value of 32,767 (inclusive).

int: By default, the int data type is a 32-bit signed two's complement integer, which has a minimum value of -231 and a maximum value of 231-1.

long: The long data type is a 64-bit two's complement integer. The signed long has a minimum value of -263 and a maximum value of 263-1.

float: The float data type is a single-precision 32-bit IEEE 754 floating point

double: The double data type is a double-precision 64-bit IEEE 754 floating point.

Take a long look at them so you know they exist. You don't need to remember their range. Complex application might use a lot of memory at run time, computing many values. Depending on the primitive used, the system will allocate a certain ammount of memory to store your variables. When a lot of them are in use, picking the right type might be essential. But most of the time, especially at the beginning, you will make use mostly of int and double. Let's give them a value with the equal sign:

int mountainHeight = 2584;

double pocketMoney = 13.34;

 

Using the “=” sign and giving them a value is called that they were “initialized”. Giving a value to a variable is called to “initialize”. Creating a new object is called to “instantiate”. Do not confuse the two terms. You will find out later that you can use a variable without initializing it from the beginning. You can declare it first and give it a value later when you need it to store a value. Remember that Java syntax is case sensitive, you can't write Int instead of int, it will give you a warning. As code style rules apply, the name of the variable has to start with lower case letter. Is a good rule of thumb to use descriptive names, easily understood. It will help you and others when many variables are in use as it makes the code more easy to follow, identify and know what the variables were designed to store. When using multi-words to name a variable, again, coding style requests the use of camel case. Words like “food for brain” will be written “foodForBrain”. Internal words will start with upper letters and end up looking like a camel's hump. Hence the word “camel case”.

Let's get back to our primitives and what to remember about them.

int will store quite a big number, probably as big as you will need at first. You don't need to worry yet about the memory usage.

double will also store a big enough number and will give you, on top, the posibility to store a decimal value to be more precise.

V. Arithmetical operators

 

 

We can use arithmetical operators the same way they are used in algebra. In Java, we have the following basic operators:

 

“+” sign for adding

“-” sign for substracting

“/” sign for dividing

“*” for multiplication

 

These operators can be used for numbers or Strings. Example:

 

int number = 4;

double moneyEarned = 3.40;

String name = "John";

String profession = "painter";

 

System.out.println(profession +" " + name + " " + "earned" + " " + moneyEarned + " " + "for" + " " + number + " " + "paintings");

 

Observe that in previous exercises we used a separate Syste.out.print() to insert space between Strings in order to make the result inteligible. With the adding sign we can concatenate several Strings adding them toghether, or we can concatenate with an empty space String to make separation between them or other type of variable.

 

To show examples of other signs we can add extra variables.

 

int number = 4;

double moneyEarned = 3.40;

String name = "John";

String profession = "painter";

String moneyPerPainting = "John earned for each painting: ";

String allMoneyEarned = "John earned for all his paintings: ";

 

System.out.println(profession +" " + name + " " + "earned" + " " + moneyEarned + " " + "for" + " " + number + " " + "paintings");

 

System.out.println(moneyPerPainting + number/moneyEarned);

System.out.println(allMoneyEarned + number*moneyEarned);

 

We use also couple of operators that you might not have been so familiar with.

 

“%” sign for modulus

“++” sign for increment

“--” sign for decrement

 

Modulus is used to divide left hand operand with right hand operand and returns remainder.

It means that if we calculate 5%3 the result is 2. That's because 3 can fit inside 5 one time so the remainder is 2. You can think of it as a divider. It will do the aritmethics between two numbers just like “/” but instead of showing you the result from dividing the two numbers, the result will be the remainder of the two. This gives you some handy applications.

For example, you can use the modulus against number 10 to get the last digit of any number:

 

int aNumber = 356;

System.out.println(aNumber%10);

 

The result will be 6. That's because dividing a number by 10 will shift a comma by one position so the remainder will be exactly the last digit. 356/10 = 35,6 Remainder = 6 Magic!

 

Another useful application of modulus is checking if a number is odd or not. If you use modulus gainst number 2 it will give you a 0 result if the number is even or a remainder is an odd number.

 

``int evenNumber = 356;

int oddNumber = 357;

System.out.println(evenNumber%2 + " " + oddNumber%2);

 

The result will be 0 and 1. Later on we will learn about an “if” clause. You can use it in combination with modulus to do a check on a number . This way, you will be able to do something depending on the 0 or 1 result.

 

Increment sign is pretty straightforward. It will increment the value of the operand by 1.

 

int number = 20;

double decimalNumber = 20.22;

number++;

decimalNumber++;

System.out.println(number + " and " + decimalNumber);

 

The two numbers were initialized by a value and then incremented with 1. Results will be 21 for int and 21.22 for double variables. Increment operator is handy when a certain variable is used as a counter or more often when needed to iterate trough an array(about which will learn later on)

 

Decrement sign is the mirror version of the increment one.

 

int number = 20;

double decimalNumber = 20.22;

number--;

decimalNumber--;

System.out.println(number + " and " + decimalNumber);

 

 

VI. Relational operators

 

 

In Java you can use the following relational operators:

 

“==” equal to

“==” will check if the value of two operands are equal or not, if yes then condition becomes true

 

As learned until now, a single “=” sign will assing a certain value. When used twice, “==” it turns intro a value check. Example:

 

int number = 10;

int secondNumber = 10;

if(number == secondNumber){//do something}

 

“!=” not equal

“!=” represents the opposite of “==” and is used to check if the value is not equal to another value.

Example:

int number = 10;

int secondNumber = 10;

if(number != secondNumber){//do something}

 

Appart from these we have the more familiar sets of operators:

 

“>” greater than if(number > secondNumber){//do something}

“<” less than if(number < secondNumber){//do something}

“>=” greater than or equal to if(number >= secondNumber){//do something}

“<=” less than or equal to if(number <= secondNumber){//do something}

 

Relational operators will be used in a certain clause and not as a stand alone operation because they imply a question, a condition in check.

 

When comparing two Strings you have to use the equals(Object anObject) method. Remember that Strings were basically an object. This method compares this String to the specified object. The result is true if and only if the argument is not null and is a String object that represents the same sequence of characters as this object. Example:

 

String car = "car";

String car2 = "car";

if(car.equals(car2)){

//do something

}

Remember that Java is case sensitive. “car “ will not be equal to “Car”.

VIII. Conditional clause “ if ”

 

Previously, we already have observed the use of an “if” clause when conditions had to be checked.

The main method, where the compiled code goes, does the calculations in a descending way. It starts from the top and goes down until the closing semicolon. When the compiler hits an “if” clause it checks for the condition. If the condition is fulfilled, it enters the body of the “if” clause and compiles the code that is inside. Similarly with checking your pokets. You tap all your pockets from the outside looking for your keys. If you feel there is something in one of your pockets you go ahead and get your keys out. (unless you have other things in your pockets then your apporach was wrong)

 

Structure of an “if” clause is as following:

 

if(condition){

//do something;

}

In other words if the condition inside the parathesis is true the compiler will go inside the curly brackets and take into account the code that lies in there. Example:

 

int number = 10;

 

if (number <10){

System.out.println("number is smaller than 10");

}

Of course number is not smaller than 10 so the compiler will not enter the body and print the statement.

Let's add another “if”:

 

int number = 10;

 

if (number <10){

System.out.println("number is smaller than 10");

}if (number ==10){

System.out.println("number is equal to 10");

}

 

This time the second condition was true and “number is equal to 10” will be printed out. Coding style suggests to write the first semicolon in the “if” clause after the condition, and the closing semicolon under the “if” This way the code is more legible. As a suggestion, is a good practice to write the opening and closing semicolon together and then press enter with the cursor between them. This way you don't get confused when you forgot the closing semicolon and you do not know where.

 

In the previous example you had 2 “if” clauses to check. Let's add another one.

 

if (number > 10){

System.out.println("number is greater than 10");

}

Of course “number is not greater than 10, it will not print the message out. Having multiple simple “if” clauses works but is not good practice. Normally you would use an “else if” clause. Let's change the code to suit best.

 

int number = 10;

 

if (number < 10) {

System.out.println("number is smaller than 10");

}

else if (number == 10) {

System.out.println("number is equal to 10");

}

else if (number > 10) {

System.out.println("number is greater than 10");

}

 

Now, it looks much better. It works the same but now, you tell the compiler like this:

 

check something, if is true. Otherwise check something else if is true. Otherwise check something else if is true … and so on... You can get as many else if clauses as you want.

 

 

When you have only two cases to check you can get rid of the second “if” and the condition parathesis because you have one condition or else, implicitly, the opposite condition. Example:

 

int number = 10;

 

if (number < 10) {

System.out.println("number is equal to 10");

}

else{

System.out.println("number is greater than 10");

}

 

In other words: if the first condition is not true than any other scenario must be true. You don't need to check a second condition, just go ahead inside the “else” statement. If the first condition would be true any other scenario is not so don't go inside the “else” statement.

 

Scanner class

 

So far we have compiled code that we specified inside the main method. But that seems a one way ticket. What can we do if we would like a user to interact with the code and calculate results according with user's input? That's why we have the Scanner class. We remember that Java has many helping classes for different scenarios that we can import when needed. We also remember that classes are blueprints for objects. To make a new object from that blueprint we need to assign it a name of our choice and tell the compiler to create a new object with that name. Let's to that.

 

Scanner ourScanner = new Scanner(System.in);

 

Scanner starts with upper case letters so it must be a class. So we create a new object named “ourScanner” from the blueprint of that class. It will be able to access the proprieties of that class with the help of the methods created in that class. We tell the compiler that “ourScanner” is a Scanner by writing “Scanner ourScanner”. Like “Phone myPhone” or “Phone johny”. We know these phones are phones because we identified them as being phones. The only thing is that we do know yet what type of phone it is. It might be a specific type of phone. So we create the copy of the phone with specific parameters in mind. In our case we create “ourScanner” with the template of a specific Scanner that has a parameter (System.in). That parameter is actually the type of constructor that the Scanner class invokes but for the sake of simplicity we will not go into more detail at this point. The Scanner class is esentially a reader. By placing the parameter “System.in” we tell the constructor of Scanner class to take the input from “System.in”

 

You already know what System.out.print() does. The Scanner will use the System class as well but this time with the instance of “in”. A bit confusing we know, but we use an object that makes use of another object as we discussed earlier.

 

So ourScanner was instantiated as a Scanner of the type Scanner(System.in). So far so good apart from the fact that out compiler does not know yet where to look from the resources of our Scanner. So we have to import it in our working class to find the refrence.

 

Write inside your main class:

 

Scanner ourScanner = new Scanner(System.in);

 

The name Scanner will be highlighted in red. Click over the name and a helper dialog will appear. Choose “import Scanner(java.util)” by clicking on it. Now Eclipse imported the package Scanner inside your own package and your class knows where to look for it.

 

We now can use ourScanner as a new Scanner object and use the methods available.

 

We know that to invoke a method from a newly created object we write the name followed by dot “.” character. Let's do that.

 

ourScanner.

 

We can see when we type “.” the suggestion dialog appeared with the methods available. We either choose from the list or type: nextLine();

As a useful tip, when chossing a method look at the long description of the method or the type of data it uses. In our case, nexLine() is returns the data into a String. If we want to use what it returns we need to store it in a String.

 

Let's create a String to store the input from the keyboard that is brought to us by the Scanner created.

 

Scanner ourScanner = new Scanner(System.in);

String input = ourScanner.nextLine();

System.out.println(input);

 

Run the code. Now the compiler is waiting from an outside input. Click on the window where normally you see the code that is run. Write something and press enter. Great! The input from you was printed out again. Lets review what we did.

 

Scanner ourScanner = new Scanner(System.in); → we imported and created a new Scanner

 

String input = ourScanner.nextLine(); → we stored what the method was returning in a String

 

System.out.println(input); → we printed out the content of our String.

 

Methods will calculate a code placed inside them. Some type will return a result of a certain type some of them will just alter already existing variables. A good practice is to check what type of method you are dealing with. If is returning something you need a valid type of variable or object to store that information and use it. You can always Google a method or a class to understand better what it can do or how much it can do.

 

Examples of using Scanner class:

 

Scanner ourScanner = new Scanner(System.in);

 

System.out.println("Please enter a name");

String name = ourScanner.nextLine(); → nextLine() returns a String

 

System.out.println("Please enter an age");

int age = ourScanner.nextInt(); → nextInt() returns an int

 

System.out.println(name + " is " + age + " years old ");
